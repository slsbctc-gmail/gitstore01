# 桌面钓鱼战斗重构设计

日期：2026-05-09

## 目标

把当前“张力安全区 + 脱钩/逃脱 + 进度条满即上鱼”的规则，重构为更接近真实钓鱼的“鱼体力 + 收线长度 + 线承压 + 爆发挣扎”模型。

## 设计原则

1. 成功条件统一为“把线收回来”，而不是单纯把进度条拉满。
2. 失败条件以“断线”为主。鱼在中钩后，只要线没断，就不应因为跑远而自然脱钩。
3. 鱼体力是拉放线互动的核心消耗对象。鱼越重，体力越强，磨鱼时间越长。
4. 好竿好线在低级场景应明显形成装备压制，小鱼可以被硬拉回来。
5. 前四个场景保留减负机制，后续场景继续保持真随机。

## 新的战斗状态

每次中鱼后维护以下临时状态：

- `FishStaminaMax` / `FishStamina`：鱼体力上限与当前值。
- `LineDistance`：当前放出去的线长，必须降到接近 0 才算上鱼。
- `Tension`：当前线张力。
- `Load`：鱼挣扎和收线共同作用下的线承压，用于判断断线风险。
- `IsBursting` / `BurstTicks`：鱼是否处于爆发挣扎期。
- `RetrieveProgress`：仅作为“收回来多少”的展示值，不再代表胜利条件。

## 战斗流程

1. 抛竿后进入等口阶段。
2. 咬口窗口到来时，成功扬竿才会进入战斗。
3. 提前扬竿：本次事件直接结束，无鱼获，不扣饵、不扣钩、不扣线。
4. 过晚扬竿：仍然视为鱼吃饵走口，扣 1 份饵料。
5. 战斗中鱼会周期性爆发挣扎。爆发期会抬高 `Tension` 和 `Load`，平稳期则适合收线。
6. 当 `LineDistance` 回收到目标阈值以下时，才真正完成捕获。

## 拉线与放线

### 拉线

- 拉线的主要效果是减少 `LineDistance`。
- 鱼不爆发时，拉线应稳定推进收线。
- 鱼在爆发时，强拉会显著提高 `Load`，但也会更快消耗鱼体力。
- 如果鱼体力低、鱼体型小、鱼线承压高，则允许形成“硬拉压制”。

### 放线

- 放线的主要效果是降低 `Tension` / `Load`，避免断线。
- 放线会让 `LineDistance` 回升，等于把之前收回来的线吐出去一部分。
- 鱼在爆发时放线，也会让鱼继续消耗体力，但效率低于有效拉线。

## 断线与钩型作用

1. 常规战斗失败只看断线，不再用脱钩作为主要失败来源。
2. 鱼钩大小和样式主要影响“中鱼概率”和“目标鱼偏好”，不再在战斗中频繁触发脱钩。
3. 鱼线只要 `Load <= MaxTension` 就不会断。
4. 当 `Load > MaxTension` 时，按超载比例和持续时间计算断线概率。
5. 切线后继续保留“损失线长 + 损失鱼钩”的后果。

## 鱼与装备参数

### 鱼新增或强化参数

- `BaseStamina`
- `BurstStrength`
- `BurstFrequency`
- `BurstDuration`
- `FatigueResistance`
- `RecommendedLineStrength`

这些参数与鱼的重量区间、稀有度、场景难度共同决定战斗强度。

### 装备作用

- 鱼竿 `Power`：提高强拉能力和硬拉阈值。
- 鱼竿 `Control`：降低爆发期的额外失控量。
- 鱼竿 `Stability`：降低被动张力波动。
- 鱼竿 `Luck`：提高稀有鱼、隐藏鱼、极品重量段概率。
- 鱼线 `MaxTension`：决定能承受多大拉力。
- 鱼钩 / 饵料：主要影响咬钩对象与中鱼率。

## 难度减负

前四个场景额外提供以下减负：

- 未收录普通鱼维持更高候选权重。
- 鱼体力和爆发强度按场景轻度下调。
- 爆发间隔更长，平稳窗口更长。

第五场景之后不做保底式战斗修正，只保留随机和装备差异。

## 幸运

幸运同时影响三件事：

1. 稀有鱼与隐藏鱼候选权重。
2. 隐藏鱼、隐藏物品的实际触发概率。
3. 生成鱼重量时更容易落在高重量段，从而提高极品概率。

## 界面最小改动

1. 新增鱼体力条。
2. 拉线/放线区展示为半圆张力表与卷线器，不再把关键状态显示在渔获区。
3. 中鱼瞬间增加画面抖动和卷线器高亮。
4. 图鉴测试视图中显示鱼偏好饵料、钩型、推荐线强度，便于验参数。

## 测试要求

自动化测试至少覆盖：

- 重鱼体力高于轻鱼。
- 鱼体力归零后，若线未收回，仍不能立刻判定成功。
- 小鱼 + 强线可以形成硬拉压制。
- 只要承压未超线强，就不会断线。
- 幸运提升隐藏、稀有、极品权重。
- 前四个场景存在减负，后续场景保持真随机。
